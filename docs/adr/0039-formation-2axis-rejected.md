# 0039. 上位近接時の2軸フォーメーションの棄却

## ステータス
棄却

## コンテキスト

現行の買い方ルール（CLAUDE.md「表記規約」）はフォーメーションを「この PJ では基本不使用」とし、
買い目は ◎1頭軸ながし or ボックスを基本とする。しかし上位人気・上位モデル評価が近接（co-favorite）
したレースでは、◎を1頭に固定すると「◎がコケたら全滅」というもろさがある。モデル勝率が拮抗した
2頭を2軸にした方が、どちらが勝っても拾えて期待回収率・的中率とも改善する可能性がある（#241）。

動機の実例（2026-06-27 福島R7）: モデル ③20.7% / ⑤19.4% がほぼ並び（市場でも単 3.3 / 3.5 の
1・2番人気）。③単軸だと③が飛んだ時に総崩れだが、③⑤2軸なら⑤×穴で拾える、という仮説。

発動条件 `prob(○) ≥ θ × prob(◎)`（θ を 0.80/0.85/0.90 で掃引、混戦判定の 0.70 より厳しめ）を
満たす近接レースに限り、baseline（◎1頭軸ながし）を以下の2軸 variant に差し替えてバックテスト比較した。
`scripts/predict-check/formation_backtest.py` を新設し、確定買い方（¥5,000・ワイド/馬連/3連複）に
機械適用した。

- **union2**（OR 意味論 / Issue の狙いに直結）: 各券種を ◎脚 / ○脚に予算折半。ワイド・馬連は
  ◎-相手 ∪ ○-相手、3連複は ◎軸ながし ∪ ○軸ながし。◎が飛んでも ○×穴で拾う。◎○両方を含む目は
  ◎脚に寄せ二重計上を避ける。
- **pair2**（AND 意味論 / 表記規約のフォーメーション定義に厳密準拠）: 全券種で ◎○両方必須。
  ワイド・馬連は ◎-○ に集中、3連複は ◎○2頭軸ながし（◎-○-相手）。

## 決定

2軸フォーメーションは採用しない。現行の **◎1頭軸ながし** を維持する。`formation_backtest.py` は
将来の再評価のために残す。

## バックテスト結果（2026-05-30〜06-21 の win オッズ・結果が揃う 127R, α=0.2, m=10）

### 発動部分集合（近接レースのみ）での比較

| θ | 発動R | variant | 回収率 | 的中率 | σ(1レースROI) |
|---|---|---|---|---|---|
| 0.80 | 40 | baseline | 65.7% | 42% | 100.9 |
|      |    | union2   | **69.1%** | 72% | 61.8 |
|      |    | pair2    | 39.2% | 15% | 102.9 |
| 0.85 | 31 | baseline | 74.9% | 48% | 107.4 |
|      |    | union2   | 69.6% | 77% | 59.7 |
|      |    | pair2    | 31.5% | 13% | 90.3 |
| 0.90 | 19 | baseline | 78.2% | 53% | 108.3 |
|      |    | union2   | 78.0% | 89% | 59.4 |
|      |    | pair2    | 33.0% | 11% | 98.6 |

### 全 127R 通し（非発動=baseline・発動時のみ差替え）の総回収率

| θ | strategy | 回収率 | 的中率 |
|---|---|---|---|
| 0.80 | baseline | 76.1% | 54% |
|      | union2   | 77.3% | 63% |
|      | pair2    | 67.8% | 45% |
| 0.85 | union2   | 74.9% | 61% |
| 0.90 | union2   | 76.1% | 59% |

（baseline は θ 非依存で全体 76.1%。pair2 は θ=0.85/0.90 でも 65.5% / 69.4% と一貫して悪化。）

## 理由

- **pair2 は全閾値で壊滅的に悪化**（部分集合 ROI 31〜39%・的中 11〜15%）。◎○の両方が上位に来ないと
  全券種が空振りする構造で、近接＝拮抗（どちらかが負けやすい）局面ほど両崩れ確率が高く、AND 集中は
  最悪の組み方だった。表記規約どおりの「フォーメーション基本不使用」を裏付ける結果。
- **union2 は回収率の改善が確認できなかった**。部分集合 ROI は θ=0.80 で +3.4pt（65.7→69.1）だが、
  隣接する θ=0.85 で −5.3pt（74.9→69.6）、θ=0.90 でフラット（78.2→78.0）と**符号が反転**する。
  全体回収率でも θ=0.80 で +1.2pt・θ=0.85 で −1.2pt・θ=0.90 でフラットと、閾値を少し動かすだけで
  優劣が入れ替わる。これは安定したエッジではなく**標本ノイズ**（発動 19〜40R と小標本）の兆候。
- union2 は的中率を 42%→72〜89% に激増させ分散をほぼ半減させるが、**回収率が伴わない**。これは
  CLAUDE.md「レース選択基準」が無価値と断ずる典型的な「高的中・低配当」。◎○を両軸に広げると当たり
  やすくなるが、当たる目が低配当の本命決着に偏り、外れ目（◎-穴 / ○-穴）に資金が分散して期待値は
  伸びない。EV/回収率を判定基準とする本 PJ の方針では採用根拠にならない。
- 動機だった福島R7（③20.7/⑤19.4, θ≈0.94）が属する最も近接した帯（θ=0.90）でも union2 は
  baseline とフラット。**動機ケースの帯でもエッジは出なかった**。

これは ADR 0030（混戦時の3連複相手拡大の棄却）と同型である。買い目を広げて的中率を上げても、
分散した資金が低配当の当たりにしか乗らず回収率が改善しない、という同じ構造。

## 影響

- CLAUDE.md の買い方ルールは変更しない（「フォーメーション基本不使用」「相手 top5・◎1頭軸ながし」を維持）。
- `scripts/predict-check/formation_backtest.py` を新設し残す（再評価用）。
- `scripts/predict-check/gen_win_backtest_data.sh` に対象期間・α・analyze バイナリパスの
  環境変数オーバーライド（`PADDOCK_BT_FROM` / `PADDOCK_BT_TO` / `PADDOCK_BT_ALPHA` /
  `PADDOCK_ANALYZE_BIN`）を追加。既定値は従来どおりで後方互換。
- θ 軸は 0.80/0.85/0.90 を評価。0.80 未満（混戦帯 0.70 へ接近）は混戦ボックスと役割が重複し、
  本 ADR の射程外とする。シグナル定義（券種ごとの配分比・相手幅）を変える場合は再スイープ余地を残す。

## 再現方法

win オッズ・結果が揃う期間でデータを生成し（analyze は別 worktree のビルドも流用可）、
formation_backtest を実行する:

```sh
# データ生成（α=0.2・対象期間を環境変数で指定）
PADDOCK_BT_FROM=2026-05-30 PADDOCK_BT_TO=2026-06-21 PADDOCK_BT_ALPHA=0.2 \
  scripts/predict-check/gen_win_backtest_data.sh /tmp/ft241

# バックテスト（θ 掃引）
python3 scripts/predict-check/formation_backtest.py \
  --races /tmp/ft241/bt_races.tsv --winodds /tmp/ft241/bt_winodds.tsv \
  --pred-dir /tmp/ft241 --results-dir /tmp/ft241 --theta-grid 0.80,0.85,0.90
```
