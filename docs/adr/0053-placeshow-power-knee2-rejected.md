# 0053. place/show 冪変換 γ≥2.5 の棄却（過補正・#305）

## ステータス

棄却（production の `EstimationConfig` は不変。place/show 校正は ADR 0047/0051 の `place_show_power=2.0` を正とする）

## コンテキスト

#305（win モデルの favorite-longshot 校正改善・純モデル value 検証）の計測中。ADR 0050（#286）は
place/show 素スコアの raw retune を棄却した際、「`place_show_power` γ≥2.5 の掃引は #286 にブロック
されない独立の最適化課題（過補正リスクの確認が要るため別途・本 ADR では未実施）」として **γ≥2.5 を
未掃引で残していた**。本 ADR はその唯一残っていた校正レバーを 4891R で掃引した結果である。

動機: production（α=0.2）の **1番人気の複勝が依然 +8.1pt 過小評価**（予測 51.6% → 実測 59.7%、
place_show_power=2.0 適用後）であり、γ をさらに上げれば本命の複勝過小評価が解けるのではという仮説。

## 検証（4891R / 2025-01-05〜2026-06-14, α=0.2・win_power=1.25, `analyze backtest`）

`paddock-analyze backtest --blend-alpha 0.2 --win-power 1.25 --place-show-power <γ>` で γ を
2.0（現行）/ 2.5 / 3.0 に振った。指定レンジ `--from 2025-01-01 --to 2026-06-30` は DB の実レースに
クリップされ評価対象は 4891R。`place_show_power` は単調変換のため複勝的中率（本命＝show 最上位の
3着内率）は全 γ で **64.0% 不変**で、効くのは校正・複勝 EV のみ。

| γ | 1番人気 複勝差 | 複勝 Brier | 複勝 LogLoss | curated place 回収率 | curated place 予測 vs 実的中 |
|---|---|---|---|---|---|
| **2.0（現行）** | **+8.1pt** | **0.1499** | **0.5858** | **65.6%** | 22.6% vs 13.0% |
| 2.5 | +3.1pt | 0.1554 | 0.7879 | 66.3% | 27.7% vs 14.4% |
| 3.0 | −0.1pt | 0.1606 | 0.9997 | 58.2% | 31.5% vs 15.7% |

（複勝差 = 実率 − 平均予測。正＝過小評価。Brier/LogLoss・回収率は小さい/大きいほど良い。）

**観察**:

- **本命の帯バイアスは γ で確かに消える**（1番人気 複勝差 +8.1 → +3.1 → −0.1pt）。人気薄の過大評価
  （10番人気以下 −7.5 → −5.9pt）も縮む。**帯単位の見かけの校正は γ とともに改善する**。
- だが**確率の質（Brier/LogLoss）は単調悪化**。複勝 Brier 0.1499→0.1606、LogLoss は 0.5858→0.9997 と
  急悪化。これは γ が分布を過シャープ化し、外した時の確信が過大になるため（LogLoss が最も鋭く罰する）。
- **curated place（複勝）買い目の回収率は改善しない**。γ=2.5 は +0.7pt（65.6→66.3、ノイズ域）、
  γ=3.0 は **−7.4pt 悪化**（65.6→58.2）。さらに買い目の予測が過信化し（γ=3.0 で予測 31.5% に対し実的中
  15.7%＝2 倍の過大予測）、**production の EV 買いに直接有害**（複勝を過大評価し過剰投資＝マイナス EV を招く）。

## 決定

place/show 冪変換 **γ≥2.5 は棄却**する。production の `EstimationConfig`（m=10 / win_power=1.25 /
**place_show_power=2.0**）は変更しない。place/show 校正は ADR 0047 採用・ADR 0051 確定の γ=2.0 を引き続き正とする。

## 理由

- **帯バイアスの解消（1番人気 +8.1→−0.1pt）は「校正の改善」ではなく「過シャープ化」**だった。γ≥2.5 は
  本命の複勝確率を持ち上げて帯平均のバイアスを消す一方、個々の予測を過信側に振り、複勝 Brier/LogLoss と
  curated place の確信過大（予測≫実的中）を悪化させる。帯バイアス指標と確率の質・回収率が乖離し、
  **賭けに効くのは後者**（複勝 EV は確率の質で決まる）。
- 複勝の **賭け回収率が改善しない**（γ=2.5 はノイズ、γ=3.0 は悪化）以上、production を過信側へ動かす
  根拠が無い。むしろ複勝を過大評価して過剰投資を招くため EV を毀損する。
- ADR 0050 が懸念した **過補正リスクが実証された**。γ=2.0 が Brier/回収・帯バイアスのトレードオフ上の
  knee（ADR 0051 で確定）であることを 4891R で再確認した。本命複勝の +8.1pt 残差は許容すべき残差。

## 影響 / 留保

- production・既存挙動とも不変。本 ADR は「place_show_power=2.0 維持」「γ≥2.5 不採用」を数値で固定する記録。
- **#305 の精度改善スコープはこれで出尽くし**。win 側は win_power（単調変換ゆえ的中・回収不変、Brier は
  γ=1.25 が最良で上げると悪化）・m/recency/form（ADR 0050 で win フラット）と合わせて改善レバーが無く、
  place/show は γ=2.0 が最良で γ≥2.5 は本 ADR で棄却。**現行モデルの確率校正に残る実改善余地は乏しい**。
- 残る未解決は **純モデルの value シグナル**（α=1.0 の想定回収率 96.4% > 純市場 73.1%, ADR 0052）の
  真偽切り分けで、これは校正（精度）ではなく賭けエッジの問題として **#272（予測フロー再設計）**に委ねる。
- 単一窓（4891R / 2025-01-05〜2026-06-14）。`place_show_power` は単調変換で複勝的中率に影響せず、過補正の
  向きは構造由来のため窓依存は小さいと判断。

## 再現方法

```sh
BIN=./target/release/paddock-analyze
PADDOCK_DB_URL=postgres://paddock:paddock@127.0.0.1:5432/paddock \
"$BIN" backtest --from 2025-01-01 --to 2026-06-30 --blend-alpha 0.2 --win-power 1.25 \
  --place-show-power 2.0      # 2.0 / 2.5 / 3.0 と振る
# 「人気帯別 複勝圏 過小評価診断」の 1番人気 複勝差、「確率校正」の複勝 Brier/LogLoss、
# 「買い目（curated）券種別 校正・回収率」の place 行（回収率・平均予測 vs 実的中率）を比較する。
```
