# ADR 0030: 混戦時の3連複「相手頭数」を広げない（バックテストで棄却）

## ステータス
承認済み

## コンテキスト
予想の買い方では「混戦」を自動判定し（◎の model 勝率の 0.70 倍以上の馬が ◎含め4頭以上 ＝ `band≥4`）、
印馬の3連複ボックスを別枠で足す（ADR 0028）。通常の3連複は **◎1頭軸ながし・相手 top5**（model 勝率
上位5頭）で買う。買い方ルールはローカル運用（手計算 + harness）で、本体コードには未取り込み。

2026-06-20 函館10R で、◎④（1着）・▲⑥（2着）を当てたが、**3着⑤が model 7番手・単勝25倍**で相手 top5 の
外におり、3連複④⑤⑥（5,110円）を取り逃した（PR #190 のレビュー中に顕在化）。そこで
**「混戦では3連複の◎軸ながしの相手を広く（top5→top7→全頭）流せば、人気薄の3着を拾えて回収率が
上がるのではないか」** という仮説が出た。一方、予算固定（混戦の◎軸ながし3連複枠 ¥1,500）では点数増で
1点が薄くなり、確率重み配分だと低 prob 組が資金から外れるトレードオフがある。

再実行可能な形で検証した。`scripts/predict-check/konsen_backtest.py` のパーサ・確定買い方を再利用し、
**混戦レースの◎軸ながし3連複の相手幅と重み付けだけ**を振った（ワイド=top3・馬連=top5・ボックス=
band[:5]・非混戦は固定で全 variant 共通 → 差分は混戦の3連複相手幅に由来）。

- 測定窓: **2026-04-19 / 05-30 / 05-31 / 06-13 / 06-14 / 06-20 の 105R（うち混戦 90R）**（DB に確率表・
  オッズが揃う全レース）。
- 予想は **market ブレンド無し（model-only）** を使用（市場オッズによるリーク回避）。相手幅の比較は
  同一確率順位への流し違いなので、as-of の漏れがあっても **variant 間の相対比較は公平**。
- 的中は netkeiba の確定払戻（組番一致）で判定。

## 決定
1. **混戦時の3連複の相手頭数は現行 top5（◎1頭軸ながし）を維持する。広げない。** 相手を top7・全頭に
   広げると **的中率は上がるが回収率は全 variant で下がった**（後掲）。
2. 重み付けを均等（equal）にしても改善しない。
3. 函館10R の取りこぼしは **単発の事象として恒久ルール化しない**（同じ広げ方を 90 レースに適用すると
   トータルで負ける）。買い方を変える改善はバックテストで改善が出てから採用する（ローカルメモリ
   `feedback_measurement_ordering` と整合）。

### バックテストによる検証（105R / 混戦90R, model-only 予想・確定払戻）

「混戦3連複的中」= 混戦レースで◎軸ながし3連複が1点でも当たった本数。

| variant（混戦の3連複相手幅・重み） | 全体ROI | 混戦ROI | 混戦3連複的中 |
|---|---|---|---|
| **baseline（top5・確率重み, 現行）** | **87.7%** | **61.0%** | 13/90 (14%) |
| top7・確率重み | 84.4% | 57.2% | 22/90 (24%) |
| 全頭・確率重み | 83.0% | 55.5% | 21/90 (23%) |
| top7・均等 | 84.7% | 57.5% | 23/90 (26%) |
| 全頭・均等 | 82.4% | 54.8% | 20/90 (22%) |

**相手を広げると的中率は狙い通り上がる（14%→22〜26%）が、回収率は全設定で下がる**（混戦 61.0%→
54.8〜57.5%、全体 87.7%→82.4〜84.7%）。広げるほど①外れ目に資金が分散し②当たり1点が薄くなるため、
的中率の増加を配当の減少が上回る。これは買い方の指針である「**高的中・低配当は張る価値が薄い／期待値で
取捨する**」（ローカルメモリ `feedback_betting_staking`）そのもので、広げる方向は EV を悪化させる。

## 理由
- 函館10R で取り逃した 3着⑤は model 7番手の25倍。これを拾うには相手を大幅に広げるか低 prob 組へ均等に
  資金を撒く必要があるが、**それを 90 レースに一律適用すると、たまに当たる穴の配当より、外れる広い目への
  資金流出が大きい**。1レースの結果（広げれば防げた）は、母集団では逆符号になる典型例。
- 予算固定（◎軸ながし3連複 ¥1,500＝最大15点/100円）では、相手を top7（C(7,2)=21点）以上に広げても
  確率重みでは低 prob 組が資金配分から外れ、均等にすると本線1点が薄くなる。どちらも回収を底上げしない。
- ボックス（band[:5]）と非混戦の買い方は全 variant 共通に固定したため、差は純粋に「◎軸3連複の相手幅」に
  由来する。

## 影響
- 買い方ルールに変更なし（3連複の相手は top5 を維持）。本体コード（`build_portfolio`）の挙動も変更なし。
- ローカルメモリ `feedback_betting_staking` に「混戦3連複の相手拡大は本バックテストで棄却（的中率↑/ROI↓）」を記録。

## スコープと限界（過大結論を避けるための明記）
- **model-only 予想は確率が平たく出るため 105R 中 90R が混戦判定**（実運用の市場ブレンド α=0.3 込みより
  混戦は多くなる）。よって **絶対 ROI は参考値**で、信頼すべきは **variant 間の相対順位**（広げる→ROI 低下）。
- 105R・6 開催日の中規模標本。ただし本決定は「複雑化（相手拡大）を入れない」という保守側の結論であり、
  エンハンス側に改善の挙証責任があるところ、全 variant で改善が出なかったため見送る根拠としては十分。
- 検証ハーネスは `konsen_backtest.py` を拡張した使い捨てスクリプトで、本 ADR には結果表を自己完結で
  載せる方針とした（恒久コードとしては残さない）。再評価が必要なら同 harness を同様に拡張して回す。
- 買い方ロジックは現状コード化されておらず（手計算 + harness）、本 ADR は本体実装の変更を伴わない方針決定である。
