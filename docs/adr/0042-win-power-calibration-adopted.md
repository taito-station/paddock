# 0042. win_prob 冪変換による穴馬1着過大評価の校正

## ステータス
採用（γ=1.25）

## コンテキスト

確率推定は win スコアをレース内で合計 1.0 に正規化し、Harville モデルで連系・着順確率を導出する
（softmax/temperature なし）。Harville は IIA 的性質から人気薄馬の「1着」確率を過大評価しがちで、
これが穴を絡めた**馬連（着順不問）の EV を馬単（着順固定）より高く**見せる一因になる（#246）。

backtest の人気帯別校正（baseline）でも、この偏りが定量的に確認できた（4891R, α=0.2・m=10）:

| 人気帯 | 平均予測 | 実測勝率 | 偏り |
|---|---|---|---|
| 1番人気 | 26.3% | 28.2% | 過小 |
| 2-3番人気 | 13.9% | 15.0% | 過小 |
| 4-6番人気 | 7.2% | 6.8% | やや過大 |
| 7-9番人気 | 3.7% | 2.9% | 過大 |
| 10番人気以下 | 1.9% | 0.8% | 過大 |

人気馬で過小・穴馬で過大という典型的な favorite-longshot バイアス。

## 決定

ブレンド後の最終 win に冪変換 `win'_i ∝ win_i^γ`（合計 1.0 へ再正規化）を掛け、人気馬を相対強調して
穴の 1 着を縮約する（`apply_win_power`, `EstimationConfig.win_power`）。backtest sweep で **γ=1.25 を採用**。
`analyze backtest --win-power <γ>` で sweep でき、`production()` は γ=1.25 を既定にする。

冪変換は単調なのでトップ選好馬（argmax）は不変＝単勝的中率・想定回収率は全 γ で同一（31.2% / 75.3%）。
変わるのは確率の大きさ（校正）と、それに依存する EV ベースの買い目選択。穴の 1 着が縮約されるほど
本命→穴の馬単（`harville_exacta`）が馬連より相対的に有利化するため、#246(B) の馬単選択（ADR 0043）と連動する。

## バックテスト結果（2025-01-01〜2026-06-30 / 4891R, α=0.2・m=10）

### 単勝校正（全頭エントリ）

| γ | Brier | LogLoss |
|---|---|---|
| 1.0 (baseline) | 0.0544 | 0.1974 |
| **1.25** | **0.0543** | **0.1954** |
| 1.5 | 0.0551 | 0.1969 |
| 2.0 | 0.0581 | 0.2081 |

γ=1.25 が LogLoss 最良（0.1974→0.1954）・Brier も最良タイ。γ≥1.5 は悪化。

### 人気帯別校正（平均予測 / 実測勝率）

| 人気帯 | 実測 | γ=1.0 | γ=1.25 | γ=1.5 | γ=2.0 |
|---|---|---|---|---|---|
| 1番人気 | 28.2% | 26.3 | 32.1 | 37.5 | 46.7 |
| 2-3番人気 | 15.0% | 13.9 | 14.7 | 15.1 | 14.9 |
| 4-6番人気 | 6.8% | 7.2 | 6.5 | 5.8 | 4.4 |
| 7-9番人気 | 2.9% | 3.7 | **2.9** | 2.2 | 1.2 |
| 10番人気以下 | 0.8% | 1.9 | 1.3 | 0.8 | 0.4 |

γ=1.25 は穴帯（4-6・7-9・10+）の過大評価を縮小（7-9 はほぼ一致、10+ は gap 1.1→0.5）。

## 理由

- **γ=1.25 が単勝 LogLoss を最良化**し（baseline 比 0.1974→0.1954）、Brier も非劣化。本 issue の核心である
  穴帯の 1 着過大評価を縮小する（10番人気以下 1.9→1.3 / 7-9番 3.7→2.9 で実測一致）。
- **トップ選好・回収率は不変**（単調変換）。校正のみ改善する低リスクな変更。
- **γ≥1.5 は棄却**: 単勝 LogLoss/Brier が悪化し、人気馬を過剰補正する（1番人気 予測 37.5%/46.7% vs 実測
  28.2%）。γ=2.0 では win_prob 肥大で単勝の EV ゲートを誤通過する買い目が急増し過信が顕著。
- 代償として γ=1.25 は 1番人気を過小（-1.9pt）から過大（+3.9pt）へ振らせるが、これは穴帯の改善
  （多数の穴馬が母数を占める）と引き換えで、集計 LogLoss は純改善。買い目側は #246(B) が「馬単は
  EV が馬連を strict に上回るときだけ・tie は馬連」という保守ゲートで過剰な馬単化を防ぐ。

## 影響

- `EstimationConfig.win_power: Option<f64>` を追加。`Default` は `None`（後方互換 no-op）、`production()` は
  `RECOMMENDED_WIN_POWER = 1.25`。predict / backtest 両経路がブレンド後に `apply_win_power` を適用。
- `analyze backtest --win-power <γ>` を追加（未指定 no-op、再 sweep 用）。`analyze predict` は production 固定。
- 回帰ガード `production_config_is_shrinkage_m10_and_recency_off` に win_power=1.25 の固定を追加。
- 仕様書 `docs/specifications/probability-estimation.md` にステップ 5 を追記。

## 再現方法

```sh
BIN=./target/release/paddock-analyze
for g in 1.0 1.25 1.5 2.0; do
  PADDOCK_LOG=warn "$BIN" backtest --from 2025-01-01 --to 2026-06-30 \
    --blend-alpha 0.2 --shrinkage-m 10 --win-power $g
done
# 単勝 Brier/LogLoss と「人気帯別（単勝校正）」を比較する。
```
