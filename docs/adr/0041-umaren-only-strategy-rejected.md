# 0041. 馬連特化（馬連 EV≥閾値）買い目戦略の棄却

## ステータス
棄却

## コンテキスト

現行の買い方ルール（CLAUDE.md「買い方ルール」）はワイド+馬連+三連複の3券種ポートフォリオ
（◎1頭軸ながし）。だが運用上 **「ROI≥100% で張れる鞍が極端に少ない」** という課題がある。
2026-06-27（3場36鞍）の手作業実測で、フルポートフォリオで ROI≥100% はわずか2鞍だった一方、
「濃い +EV 馬連（EV≥1.2）」を1点以上持つ鞍は約14鞍あった。

仮説（#250）: **+EV は馬連に集中しているのに、ワイド（構造的低EV）と三連複（高分散）が全体EVを
希薄化し、張れる鞍を潰している**。馬連に絞れば −EV 閾値を下げずに張れる局面（frequency）を
増やせるのではないか。

ただし #250 自身が「+EV 馬連の多くは穴絡みの高オッズ・低的中で、モデルEVが過大に出やすいゾーン
（#246）。額面EVでの単発買いはノイズを掴むリスクがある」と警告しており、feel で採用せず
バックテストで検証することを条件としていた。本タスクのスコープは**馬連のみ**（馬単は #246 の
較正後に別途。穴の馬単 EV はノイズ掴みリスクが高いため）。

`scripts/predict-check/umaren_backtest.py` を新設し、◎1頭軸ながし（軸=◎の1頭固定, #241 準拠。
相手は CLAUDE.md の top5 に限らず**全頭**から抽出する。+EV は穴に偏り top5 では捕れないため）のうち
馬連 model EV（`p_top2_set(◎,相手) × 馬連オッズ`）が閾値 θ 以上の点のみを買う戦略を、
配分（flat=均等 / weighted=馬連的中確率重み）・閾値（θ=1.0/1.2/1.5）・オッズ上限（cap=∞/50/30/20）で
掃引して比較した。

### 検証設計上の核心2点

- **EV フィルタと払戻清算でオッズ源を分離した（循環回避）**。EV≥θ のフィルタに使う DB 盤面オッズ
  （`race_odds`）と、的中時の払戻を同じオッズで清算すると「EV≥θ の点は当たれば必ず θ 倍返る」
  恒真化で本戦略が不当有利になる。よって **フィルタ＝DB 盤面オッズ／清算＝実払戻（netkeiba
  result.html の実配当）** と分離した。これによりモデル確率の較正不良（穴過大評価, #246）が
  実 ROI に正しく反映される。
- **対照 baseline_pf は wide を含まない**。馬連・三連複の盤面オッズは結果ありレースで全 71R 揃うが、
  wide オッズは履歴欠落（fetch_wide.py が直近ライブ用に取った別レース群にしか無く、結果ありレースと
  交差ゼロ）。#250 の主張は「wide が EV を希薄化」なので、wide 抜き baseline はむしろ**強い対照
  （steel-man）**。新戦略が wide 抜き baseline すら上回れなければ結論は頑健。

## 決定

馬連特化（馬連 EV≥閾値）戦略は採用しない。現行の3券種ポートフォリオ（◎1頭軸ながし）を維持する。
`umaren_backtest.py` は将来の再評価（特に #246 較正後）のために残す。

## バックテスト結果（2026-05-30〜06-14 の 71R, α=0.2・m=10）

発動部分集合（＝実際に賭けた鞍）のみで集計。回収率の分母は実張額。判定は ROI / 的中 / σ（スケール
不変）で行う。maxDD(円) は戦略間で予算が異なる（baseline_pf ¥3,500 / top5 ¥1,500 / 全頭 flat・umaren
¥5,000）ためスケール非不変＝直接比較不可（参考列）。

| 戦略 | 張鞍/71 | ROI | 的中率 | σ(1R ROI) | maxDD(円,参考) |
|---|---|---|---|---|---|
| baseline_pf（馬連+三連複−wide, model ROI≥100% ゲート）| **1** | 196.6% | 100%(n=1) | 0.0 | 0 |
| 参考: 馬連 ◎軸 top5 ながし（無フィルタ・¥1,500）| 71 | **79.0%** | 42% | 107.2 | 23,590 |
| 参考: 馬連 ◎軸 全頭ながし（無フィルタ・flat・¥5,000）| 71 | **66.4%** | 52% | 125.1 | 119,310 |
| 馬連特化 θ=1.0 cap∞（#250 素案）| 71 | 43.7% | 4% | 255.9 | 224,470 |
| 馬連特化 θ=1.2 cap∞ | 71 | 49.9% | 1% | 417.3 | 260,000 |
| 馬連特化 θ=1.5 cap∞ | 68 | 0.0% | 0% | 0.0 | 340,000 |
| 馬連特化 θ=1.0 cap50 | 28 | 43.2% | 4% | 224.5 | 80,000 |
| 馬連特化 θ=1.0 cap30（規律版で最良）| 13 | 93.1% | 8% | 322.4 | 35,000 |
| 馬連特化 θ=1.0 cap20 | 5 | 242.0% | 20% | 484.0 | 15,000 |
| 馬連特化 θ=1.2 cap30 | 4 | 0.0% | 0% | — | 20,000 |

（flat と weighted はほぼ同値。cap∞ weighted は θ=1.2 で 57.5% と最高だが的中 1%で実態は同じ。
cap20/θ=1.0 の 242% は 5 鞍のみの小標本ノイズ。全行は `umaren_backtest.py --cap-grid inf,50,30,20` の
出力を参照。）

## 理由

- **馬連特化は frequency を 1→71 鞍に激増させるが、それは見せかけ**。EV フィルタは全レースで
  極端な穴（馬連オッズ 76〜720 倍・モデル勝率 0.6〜2%）を選抜する。`0.6% × 720倍 = EV4.0` のように、
  モデルが穴の勝率を過大評価しているため（#246）ほぼ全鞍で「偽の +EV」が量産される。張れる鞍が
  増えるのは −EV ノイズを買っているだけで、realized ROI は 44〜58%（θ=1.5 では 68 鞍で的中 0・ROI 0%）
  に崩壊する。**「張れる鞍を増やす」目的は達成されるが無価値**（CLAUDE.md「高的中・低配当は無価値」の
  逆パターン＝低的中・偽高配当期待）。
- **EV フィルタは無フィルタより realized ROI を悪化させる＝逆予測的**。これを交絡なく示すため、
  universe（全頭）と配分（flat・¥5,000）を umaren_only と完全に揃え、**EV フィルタの有無だけ**を変えた
  apples-to-apples 対照「馬連 全頭ながし 無フィルタ flat」を置いた。結果は **無フィルタ 66.4%（的中 52%）
  → EV≥1.0 フィルタ 43.7%（的中 4%）** と、フィルタを入れるほど悪化する。つまり「相手範囲を top5→全頭に
  広げたこと」ではなく **model EV ランキングそのものが逆予測的**（高 EV と判定した点ほど当たらない）。
  さらに堅めの ◎軸 top5 無フィルタ（¥1,500）は 79.0%・的中 42% で、どの馬連特化構成よりも良い。
  閾値を上げる（θ=1.5）ほど純粋に穴の尾だけが残り的中 0% になる。
- **オッズ上限で穴を除外しても ROI≥100% を意味ある鞍数で満たす構成は無い**。cap30/θ=1.0 が
  最良（13 鞍・ROI 93.1%）だが 100% に届かず、cap20/θ=1.0 は 242% だが 5 鞍の小標本ノイズ。
  cap を絞るほど発動鞍が減り、#250 の主目的（frequency 増）と両立しない。
- **根本原因は券種選択ではなくモデルの較正不良（#246）**。+EV が馬連に「集中して見える」のは穴の
  勝率過大評価の副作用であり、馬連に絞っても較正不良の上に張るだけ。frequency 問題は #246 の較正を
  先に解決しない限り、券種ポートフォリオの組み替えでは解けない。

これは ADR 0033（条件付き単勝の棄却: 単勝市場がモデル win prob と高相関でエッジが出ず）と同型で、
「額面 model EV の閾値抽出は較正不良ゾーンでノイズを掴む」という同じ構造である。

## 検証上の留保（結論には影響しない）

- **baseline_pf の発動 1 鞍は n=1**。ROI 196.6%・的中 100% は単一の偶然で統計的意味はない。
  ここで示したいのは「現行ルールの ROI≥100% ゲートが 71R で 1 鞍しか通さない」という frequency 課題の
  再現であり、その絶対 ROI ではない。
- **wide を baseline から除外**（履歴オッズ欠落）。ただし #250 の主張に照らせば wide 抜きは強い対照で
  あり、馬連特化が勝てない結論は揺るがない（むしろ wide を足せば baseline はさらに張れなくなる方向）。
- **2026-06-27（仮説を生んだ当日）は母集団に含めない**（look-ahead / 過学習回避）。
- **フィルタ源と清算源の独立性は「盤面スナップショット→締切までのオッズドリフト分」に限る**。馬連は
  パリミュチュエルで盤面オッズと最終配当がほぼ一致するため、循環は完全には消えない。ただし残存する
  相関は本戦略を**実際より良く見せる方向**であり、それでも崩壊する以上、棄却結論はより頑健になる。
- **馬単はスコープ外**（#246 較正後に別途検証）。本 ADR の棄却は馬連特化に限る。

## 影響

- CLAUDE.md の買い方ルールは変更しない（3券種ポートフォリオ・◎1頭軸ながし・相手 top5 を維持）。
- `scripts/predict-check/umaren_backtest.py` を新設し残す（再評価用。馬単対応や #246 較正後の再走に流用）。
- frequency 課題（張れる鞍が少ない）は #246（穴の1着過大評価の較正）に依存することが定量的に裏付けられた。
  券種選択での解決は #246 完了後に再検討する。

## 再現方法

馬連・三連複の盤面オッズと結果が揃う期間でデータを生成し、umaren_backtest を実行する:

```sh
# [1] predict + 単勝オッズ + 結果 HTML（α=0.2）。res HTML は既存を流用可。
PADDOCK_BT_FROM=2026-05-30 PADDOCK_BT_TO=2026-06-14 PADDOCK_BT_ALPHA=0.2 \
  PADDOCK_ANALYZE_BIN=/path/to/target/release/paddock-analyze \
  scripts/predict-check/gen_win_backtest_data.sh /tmp/bt250

# [2] 馬連・三連複の盤面オッズを DB からエクスポート
psql "$DB" -tA -F$'\t' -c "
  SELECT o.race_id, o.bet_type, o.combination_key, o.odds::text
  FROM race_odds o JOIN race_cards rc ON rc.race_id=o.race_id
  WHERE o.bet_type IN ('quinella','trio')
    AND rc.date >= '2026-05-30' AND rc.date <= '2026-06-14'
  ORDER BY o.race_id, o.bet_type, o.odds;" > /tmp/bt250/bt_exotic_odds.tsv

# [3] バックテスト（θ × cap × flat/weighted 掃引。cap20 も含めて本 ADR の全行を再現）
python3 scripts/predict-check/umaren_backtest.py \
  --races /tmp/bt250/bt_races.tsv \
  --pred-dir /tmp/bt250 --results-dir /tmp/bt250 \
  --exotic-odds /tmp/bt250/bt_exotic_odds.tsv --ev-grid 1.0,1.2,1.5 --cap-grid inf,50,30,20
```

## 再評価（#252・#246 較正後）

ステータスは **棄却を維持**する。

本 ADR は棄却の根本原因を「モデルの穴1着過大評価（#246）で馬連 model EV が逆予測的」と特定し、
「#246 較正後に再走する」ことを条件としていた。#246（穴1着確率の冪変換較正・ADR 0042）が merged
（PR #256）されたため、**同一窓（2026-05-30〜06-14 の 71R）・同一設計**で較正後モデルを再ビルドして
再走した（#252）。較正が効いていることは、全セルが較正前テーブルと異なる値になったこと
（例: baseline_pf 196.6%@1R→24.5%@13R、umaren θ=1.0 cap∞ 43.7%→29.1%）で確認した。

### 較正後バックテスト結果（71R, α=0.2・m=10。flat。weighted もほぼ同値）

| 戦略 | 張鞍/71 | ROI | 的中率 | σ(1R ROI) |
|---|---|---|---|---|
| baseline_pf（馬連+三連複−wide, model ROI≥100% ゲート）| 13 | 24.5% | 15% | 58.2 |
| 参考: 馬連 ◎軸 top5 ながし（無フィルタ・¥1,500）| 71 | **78.7%** | 42% | 104.7 |
| 参考: 馬連 ◎軸 全頭ながし（無フィルタ・flat・¥5,000）| 71 | **66.4%** | 52% | 125.1 |
| 馬連特化 θ=1.0 cap∞ | 70 | 29.1% | 7% | 166.5 |
| 馬連特化 θ=1.2 cap∞ | 69 | 2.5% | 1% | 20.2 |
| 馬連特化 θ=1.0 cap50 | 52 | 52.2% | 8% | 209.1 |
| 馬連特化 θ=1.2 cap50 | 14 | 86.4% | 7% | 311.6 |
| 馬連特化 θ=1.0 cap30 | 48 | 59.2% | 8% | 218.9 |
| 馬連特化 θ=1.2 cap30 | 10 | 121.0% | 10% | 363.0 |
| 馬連特化 θ=1.0 cap20 | 43 | 61.2% | 7% | 232.7 |
| 馬連特化 θ=1.2 cap20 | 6 | 201.7% | 17% | 450.9 |

（θ=1.5 は全 cap で的中 0%・ROI 0%。weighted は flat と±数 pt で結論不変。）

### 棄却を維持する理由

- **逆予測は較正後も解消していない（決定的）**。universe（全頭）と配分（flat）を揃え EV フィルタの
  有無だけを変えた apples-to-apples 対照で、**無フィルタ 66.4% → EV≥1.0 フィルタ 29.1%** と
  フィルタを入れるほど悪化する。較正前（66.4%→43.7%）より**むしろ悪化**しており、model EV ランキングが
  高 EV と判定した点ほど当たらない構造は #246 較正では解消しなかった。
- **>100% を満たすのは小標本・cap 限定アーティファクトのみ**。θ=1.2 cap30（10鞍 121.0%・的中 10%・
  σ363）・cap20（6鞍 201.7%・σ451）が 100% を超えるが、的中 1〜2 鞍の高分散小標本。較正前に θ=1.0 で
  出ていた同型アーティファクト（cap30 13鞍 93.1% / cap20 5鞍 242%）が θ=1.2 へスライドしただけで、
  「frequency（張鞍）× ROI」を両立する頑健な構成は無い。cap を絞るほど発動鞍が減る制約も不変。
- **堅め対照に勝てない**。◎軸 馬連 top5 無フィルタ（71鞍 78.7%）を意味ある鞍数で上回る馬連特化構成は
  無い（θ=1.0 系は 52〜61%）。EV フィルタは依然として無フィルタ・無作為より劣る。

→ #250 の主目的「−EV 閾値を下げず張鞍を増やす」は較正後も無価値（低的中・偽高配当）であり、券種を
馬連に絞る戦略は採用しない。現行 3 券種ポートフォリオ（◎1頭軸ながし・相手 top5）を維持する。

### 副次的発見（本 ADR の射程外・要フォロー）

較正後、baseline_pf の発動が **1鞍/196.6% → 13鞍/24.5%** に激変した。較正（穴縮約＝人気馬確率の上昇・
再正規化, ADR 0042）で**人気馬偏重のポートフォリオが model ROI≥100% ゲートを多く通過するが、実 ROI は
24.5%**（無フィルタ top5 の 78.7% を大きく下回る）。すなわち較正後の model EV ゲートは「低オッズ人気馬で
EV≥100% に見えるが実配当が伴わない鞍」を通しやすい。これは **CLAUDE.md のライブ ROI≥100% ゲートの
含意**であり、馬連特化の可否（本 ADR）とは別問題として切り出し、**#263** で検討する。

なお #252 のもう一つのスコープ外項目「馬単（exacta）特化の 71R バックテスト検証」（順序プレミアムが
実エッジか。馬単は ADR 0043 でライブ採用済みだが 71R 未検証）は **#262** に分離した。
