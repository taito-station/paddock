# ADR 0033: 条件付き単勝（単勝EV≥閾値の馬だけ）を追加しない（バックテストで棄却）

## ステータス
承認済み

## コンテキスト
現行の買い方は **3券種（ワイド¥1,500/馬連¥1,500/3連複¥2,000）の◎1頭軸ながし**（合計¥5,000）。
単勝は「高的中・低配当」に終わりやすく EV が薄いと判断して対象外にしてきた。

「model の win 確率 × 単勝オッズ ≥ 閾値（＝期待値 EV）を満たす馬だけ単勝を追加買いすれば
プラス EV を拾えるのではないか」という仮説が出た（Issue #208）。具体的には

- 条件: `model_win_prob / 100 × win_odds ≥ EV_threshold`（閾値 = 1.0 / 1.1 / 1.2）
  　`model_win_prob` は %スケール（0〜100）なので /100 で実数化してから `win_odds`（払戻倍率）を掛ける
- 発動馬 1 頭あたり ¥500 の枠を均等分配（最大 5 頭まで）
- 予算モード: **add**（3券種 ¥5,000 に ¥500 を上乗せ）と **split**（3券種を按分縮小して ¥500 を捻出）の 2 種

再実行可能な形で検証した。`scripts/predict-check/gen_win_backtest_data.sh` でデータを準備し、
`scripts/predict-check/win_backtest.py` でバックテストを実施した。

- 測定窓: **2026-05-30 / 05-31 / 06-13 / 06-14 の 71R**（DB に確率表・単勝オッズ・結果が揃うレース全件）
- 予想は **market ブレンド α=0.3**（通常運用と同一設定）。
- 的中は netkeiba の確定払戻で判定。
- 回収率（ROI）の分母は実消化額。

## 決定

**条件付き単勝は追加しない。現行 3 券種構成を維持する。**

全閾値・全予算モードで baseline（3 券種のみ）を下回った。

### バックテスト結果（71R / α=0.3 / 確定払戻）

| variant | 回収率 | 差分（対 baseline） | 単勝発動 |
|---|---|---|---|
| baseline（3 券種のみ） | **79.5%** | — | — |
| +win≥100%/add | 76.0% | −3.5pt | 344 回 |
| +win≥100%/split | 74.5% | −5.0pt | 344 回 |
| +win≥110%/add | 76.0% | −3.5pt | 340 回 |
| +win≥110%/split | 74.5% | −5.0pt | 340 回 |
| +win≥120%/add | 76.0% | −3.5pt | 327 回 |
| +win≥120%/split | 74.5% | −5.0pt | 327 回 |

## 理由

1. **発動頻度が高すぎる**: 71R で 327〜344 回（1レース平均 4.6〜4.9 頭）が閾値を超えた。
   モデルが EV≥100% と判定した単勝が手当たり次第に発動しており、「絞り込みの効果」が出ていない。
2. **単勝市場の効率性**: モデルの win prob は市場の単勝オッズと高度に相関する（ADR 0027 参照）。
   「model × odds ≥ 1.0」を満たすには model が市場より高い win prob を付けた馬を選ぶが、
   この差分は真のエッジを示さず単なるモデル誤差のノイズに過ぎなかった。
3. **split モードの追加悪化**: 3 券種の予算を単勝に振り向けると本線の賭金が薄くなり、
   ワイド・馬連・3連複の当たりが細くなる。add より概ね 1.5pt 悪い（add −3.5pt / split −5.0pt）。
4. **閾値を上げても改善しない**: 100%/110%/120% いずれも同じ回収率（add −3.5pt）で差がない。
   閾値 EV でのスクリーニングはこのモデル・オッズ組合せでは機能しなかった。

## 影響

- 買い方ルールに変更なし（3 券種構成を維持）。
- ローカルメモリ `feedback_betting_staking` に「単勝 EV≥閾値の条件付き単勝は本バックテストで棄却」を記録。

## スコープと限界

- 71R（4 開催日）の標本。単勝は払戻分散が大きいため N が小さいが、
  本決定は「追加しない」という保守側の結論であり、全閾値・全モードで改善が出なかったため棄却根拠として十分。
- モデルは win prob を `blend-alpha 0.3`（市場 30% ブレンド）で算出。
  モデルが市場と完全に独立な予測力を持つ場合は異なる結果になりうるが、
  現行モデルの精度（ADR 0027 参照）では確認できなかった。
