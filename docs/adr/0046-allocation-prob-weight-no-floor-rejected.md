# 0046. 配分の確率重み化＋脚ごと最低¥100撤廃（71R で実 ROI 悪化・棄却）

## ステータス

棄却（CLAUDE.md・コードの配分は現状の均等割りを維持。確率重み化＋最低¥100撤廃は採用しない）

## コンテキスト

#272 の買い方見直しで、配分（券種内の各脚への賭け金の振り分け）の解釈が論点になった。

- ユーザー指摘: CLAUDE.md「各点の金額は組合せの確率で重み付け」「薄い相手も最低 ¥100 置いて取りこぼしを
  防ぐ」のうち、**「¥100 は馬券の額面単位であって、薄い脚に必ず ¥100 を置けという意味ではない」**。
  会話初期にはこの「薄い相手も最低¥100」の spread が **EV 希釈の主因**ではないかとも疑われていた。
- 現状の乖離: 実装コードの券種内配分は**均等割り**（`distribute`）で、CLAUDE.md が謳う「確率重み＋最大剰余」
  とも一致していない（どちらも実 ROI は後述のとおり同等）。

そこで「券種内配分を**真の的中確率で重み付け**し、**脚ごとの最低¥100（spread）を撤廃**して薄い脚は ¥0
（買わない）にする」案を検証した。総賭金は方式に依らず一定（券種予算ちょうど）なので、差は純粋に
「どの脚に厚く張るか」だけに由来する。

## 検証設計

`umaren_backtest.py` の baseline ポートフォリオ（馬連 ◎軸ながし top5 ¥1500 + 三連複 ◎軸ながし top5
¥2000、wide 除外）を土台に、券種内配分方式だけを変えて 71R（2026-05-30〜06-14, /tmp/bt252,
α=0.2・m=10）の実 ROI を比較した。全鞍機械買い・ゲート無し（配分効果の分離）。清算は netkeiba 実払戻。

- 重み基準: `proxy`（勝率: 馬連=probs[p] / 三連複=probs[a]*probs[b]）vs `true`（真の的中確率:
  馬連=P(軸·相手が上位2着) / 三連複=P(3頭が上位3着), simulate 由来）vs `equal`（均等）。
- 最低口数 `minu`: 1（脚ごと最低¥100）vs 0（撤廃・薄い脚は¥0）。

## バックテスト結果（同一ポートフォリオ・総賭金一定・71R）

| 配分方式 | 実 ROI | 的中率 |
|---|---|---|
| proxy 重み + minu=1（**現行ルール相当**）| **75.5%** | 48% |
| equal 均等 + minu=0（**現行コード実挙動**）| **75.5%** | 48% |
| true 的中確率重み + minu=1 | 71.1% | 48% |
| proxy 重み + minu=0（最低¥100 撤廃のみ）| 66.4% | 48% |
| true 的中確率重み + minu=0（**検討した変更**）| **66.3%** | 48% |

- 最低¥100 の撤廃だけで **−9pt**（75.5→66.4）、重み基準を真確率にするだけで **−4pt**（75.5→71.1）。
  両方やった検討案が最悪（66.3%）。的中率は全方式 48% で不変（総賭金一定・賭ける鞍も同一のため）。

## 決定

**配分は現状（均等割り）を維持し、確率重み化＋脚ごと最低¥100撤廃は採用しない。**

## 理由

- **薄い脚への少額 spread は実 ROI を上げている**。三連複の薄い穴目への ¥100 ずつが「たまに来る高配当を
  拾うミニボックス」として機能し、本命集中（低配当寄せ）に変えると一発を取り逃す。ADR 0030（相手拡大で
  回収悪化）の裏返しで、こちらは「薄い脚を削ると回収悪化」。
- 「¥100 は額面単位であって脚ごとの下限ではない」という**解釈は正しい**が、それを「薄い脚は¥0」と実装
  すると**この窓では負ける**。解釈の正しさと回収率が逆を向く。
- 現行コードの均等割り（75.5%）と CLAUDE.md の確率重み＋¥100（75.5% 相当）は実 ROI 同等。**変える動機が
  回収率側に無い**。

## 影響

- コード: `src/domain/src/portfolio/mod.rs` の券種内配分は均等割り（`distribute`）を維持。本 #272 では
  各買い目の**的中確率 `hit_prob` を算出・表示**する（判断材料）のみ追加し、配分は不変。
- CLAUDE.md: 配分ルールは変更しない。
- `docs/specifications/betting-rule-history.md` に棄却記録として追記。

## 検証上の留保（結論の向きには影響しない）

- **単一 71R 窓**。ただし「集中（minu=0）にするほど実 ROI が下がる」方向は proxy/true 両基準で一貫。
- wide 除外（履歴オッズ欠落。ADR 0041 と同じ steel-man 対照）。
- 無ゲート universe 自体が 75.5% < 100%＝この窓にモデルの実証エッジは無い（ADR 0044 と同様）。本件は
  その universe 内での配分方式の相対比較。
