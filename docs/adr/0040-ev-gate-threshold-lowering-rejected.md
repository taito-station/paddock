# 0040. EV ゲート閾値（ROI≥100%）引き下げの棄却と較正検証の保留

## ステータス
棄却（閾値引き下げ）／較正検証は #249（要 #248 のデータ）で保留

## コンテキスト

「+EV(ROI≥100%) の買い目が全然来ない。方針が合っているのか不安」という問題提起を起点に、
2026 年に +EV レースが実際どれくらいあったかを調査した（2026-06-27 の調査セッション）。

DB に保存済みのオッズと `scripts/predict-check/live_ev.py`（買い目＝ワイド・馬連・3連複の3券種
ROI = 期待回収率）で、保存データから再評価した。+EV 判定は結果を使わず、発走前オッズ × モデル
勝率（Plackett-Luce）の **期待 ROI** で行うため、オッズと予想確率さえあれば事後に再計算できる。

DB の在庫は 2026 年で実質 8 開催日・161R。うち**4券種オッズ（単勝＝α ブレンド入力／ワイド・馬連・
3連複＝買い目）が揃って厳密に EV 評価できたのは 2026-06-21・06-27 の 69R のみ**だった
（ワイドは `race_odds` 最新値に直近2開催しか残らず、
時系列 `race_odds_snapshots` も 06-27 分しか無い。データ保持の課題は #248）。

この調査結果を受けて、買い方ルールの EV ゲート閾値（CLAUDE.md「レース選択基準」の ROI≥100%）が
厳しすぎないか＝閾値の妥当性を再検証した。

## 決定

**EV ゲート閾値 ROI≥100% を維持する。閾値の引き下げは採用しない。**
推定 ROI の較正検証（予測 ROI が実現 ROI の偏りのない推定か）は、検証に必要なデータが現状 DB に
無いため実施できず、#248（最終オッズの snapshot 保存）でデータが揃ってから #249 で行う。

## 理由

- **ROI≥100% は損益分岐そのもの**。ROI = 期待払戻 ÷ 賭金 で、≥100% は「期待値プラス」の定義。
  閾値を 90% 等に下げる＝**期待値マイナスのレースを承知で買う**ことで、長期では必ず負ける。
  閾値は自由に動かせるツマミではなく、エッジの有無を分ける境界そのもの。
- **平均 ROI が券種フェア値とほぼ同水準（やや負側）＝モデルに平均的なエッジは無い**（ここでの
  「較正」は「市場と平均的にほぼ等価」の意味であり、予測 ROI が実現 ROI の偏りのない推定かという
  真の較正＝下記の #249 は別問題で未検証）。厳密評価できた 69R の予測平均 ROI は **73.1%**。本戦略が
  使う券種の JRA 控除率（ワイド・馬連 22.5%／3連複 25%）が示すフェア ROI ≈ **75〜77.5%** を
  数 pt 下回り、平均的なエッジは無い（むしろ僅かに負側）。+EV は控除率を食い破るほど
  市場が歪んだ稀なレースだけに出る。**構造的に稀**であり、買い目が来ないのは機能不全ではなく
  ゲートが正しく −EV を弾いている証拠。
- **ゲートが過剰に弾いている兆候は無い**。
  - 予測側: 厳密評価 69R（ワイド込み, α=0.2）の ROI 分布は 平均 73.1% ／ 最高 97% ／ +EV(≥100%) 0本
    （90〜99% が 4本、80〜89% が 17本、70〜79% が 22本、70% 未満が 26本）。α=0.3 でも +EV 0本で変わらず。
  - 実現側: 結果のある 71R の実現 ROI は `konsen_backtest.py` baseline で **83.9%**（box-off variant は 84.0%）（ADR 0028）。
  - 予測・実現とも 100% を割っており、閾値直下に「本来買うべきレース」が溜まっている形跡は無い。
  - 弱い示唆として実現(84%)>予測(73%)。推定がやや保守的な可能性はあるが、両者は別レース群・別買い目
    構造（実現側はワイド欠落）で**同一レースのペア比較ができていない**ため確定できない → 較正の宿題（#249）。
- **較正の本検証に足る標本がまだ無い**。較正を厳密に測るにはワイド込みオッズと結果が同居するレースが
  必要だが、**DB 単一ストアでは同居しない**——`race_odds` でワイドがあるのは 06-21・06-27 のみで、
  この2日は DB `results` 表が空（古い 05-30〜06-14 は DB に結果があるがワイドが無い）。
  ただし結果自体は netkeiba から取得できる（ADR 0039 / #247 の 127R backtest が実際にそうしている）ため、
  **06-21 の 35R（ワイド=DB＋結果=netkeiba 取得可）だけは予備的な較正が技術的には可能**。しかし
  単日・35R の小標本で、しかも 06-21 のオッズは午後取得＝確定前のため、ゲート位置を動かす確証には足りない。
  確証ある較正は #248 で全開催の最終オッズ snapshot を積んでから #249 で行う。本 ADR では数字を確定させない。

## 検証上の留保（結論には影響しない）

- **保存オッズの鮮度**: 06-27 のオッズは本日朝（09〜12時 JST）取得で確定前、06-21 は午後取得で
  ほぼライブ。発走直前の最終オッズではないため、最終的な +EV 本数は厳密値ではない（より確定に近い
  オッズで僅かに変動しうる）。最終オッズの定常保存は #248。
- **過去ライブ +EV は再現不可**: 古い開催の保存オッズは当時のライブ値と一致しない。例えば
  2026-06-20 函館12R は履歴ログでは ◎④1.7倍 / model 35% / ROI≈125%（当時のライブで +EV 判定）だが、
  DB は ◎④2.7倍 / model 25% で再現できない。よって「2026 年の年間 +EV 本数」を保存データから
  厳密集計することは現状できず、記録に残るライブ +EV は 06-20 の1本のみ。

## 影響

- CLAUDE.md「買い方ルール／レース選択基準」の **ROI≥100% は変更しない**。
- 較正検証（予測 ROI バケット × 実現 ROI）は **#249** で実施。前提データは **#248** が供給する
  （ブロッカー: #248）。
- 本調査の再評価スクリプト・中間データは再現用に保持（リポジトリ未コミット）。`live_ev.py` を
  そのまま過去 snapshot に当てれば、#248 蓄積後に年間 +EV 発生率を厳密に出せる。

## 再現方法

保存済みオッズ（再 fetch なし）に `analyze predict` と `live_ev.py` を当てて全3券種 ROI を出す:

```sh
# 各開催日について DB から horses / exotic / wide / meta TSV を構築し、
# analyze predict --blend-alpha 0.2 で確率表（pred.txt）を生成して live_ev.py に渡す。
# （netkeiba 再 fetch はしない＝保存済み race_odds のみを使用）
python3 scripts/predict-check/live_ev.py \
  --pred pred.txt --meta meta.tsv --horses horses.tsv \
  --exotic exotic.tsv --wide wide.tsv --budget 5000
```

ワイドが揃うのは現状 06-21・06-27 のみ。それ以外の日はワイド欠落で ROI が過小評価される
（買い目の一部に配当が無く分子に寄与しないため）。
