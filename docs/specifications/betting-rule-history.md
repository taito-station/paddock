# 買い方ルールの決定根拠・棄却記録・バックテスト履歴

現行の買い方ルールが「なぜこの形なのか」を残すアーカイブ。**運用ルール本体ではない**ので予想実行時に読む必要はなく、ルール変更を検討する時にだけ参照する。

- **現行ルール本体**: プロジェクト `CLAUDE.md` の「買い方ルール」セクション（毎セッション読込）。
- このドキュメントは、過去に検討して**棄却した改善案**と、ADR に未収録の**バックテスト/実績ログ**を保管する。棄却済みの案を再提案して再検証する無駄を防ぐのが目的。

## 棄却記録

### ① 混戦判定にオッズ条件を追加（棄却 / 2026-06-20 / ADR 0028）

「◎の単勝が X 倍以上なら混戦扱いにする」案。混戦判定にオッズ条件を足すと回収が改善するかを検証した。

- 検証: 71R（2026-05-30, 05-31, 06-13, 06-14）を `scripts/predict-check/konsen_backtest.py` で評価。
- 結果: **box-off（混戦なし）84.0% ≧ baseline（band≥4）83.9%**。オッズ条件は baseline を上回る閾値が一つも無かった。
- 追加発動した閾値（odds≥3.0/3.5 × band≥2/3）はいずれも回収を悪化させ **79.5〜81.5%**。唯一 odds≥4.0 は葵S（◎3.8倍）が閾値未満で追加発動 0 件 → **baseline 83.9% に退化（改善なし）**。band≥2 まで緩めると 13〜20R を混戦化し −3〜4pt。動機だった葵S は band=2 でボックスが組成不能（3頭未満）となり ¥1,500 が空振り。
- **結論**: オッズ条件は入れない。現行の `band≥4`（model 勝率 0.70 倍以上が ◎含め 4 頭以上）を維持。エンハンスは「回収を改善する証拠」が出てから入れる。
- 詳細: [docs/adr/0028-konsen-odds-trigger-rejected.md](../adr/0028-konsen-odds-trigger-rejected.md)（関連 Issue: #180）

### ② 混戦時の 3連複の相手拡大（top5 → top7/全頭）（棄却 / 2026-06-20 / ADR 0030）

混戦時に 3連複の相手を top5 より広げる案。函館10R（2026-06-20）で ◎④1着・▲⑥2着・3着⑤（model 7番手/25倍）を取り逃したのが動機。

- 検証: 105R（うち混戦 90R）。
- 結果: 的中率は 14%→22〜26% に向上するが、**回収率は全設定で低下**。
  - 混戦 ROI: 61.0% → 54.8〜57.5%
  - 全体 ROI: 87.7% → 82.4〜84.7%
- 均等重みでも改善せず。広げるほど外れ目に資金が分散し、当たり 1 点が薄くなる（典型的な「高的中・低配当」）。
- **結論**: 現行 top5 を維持。単発の取りこぼしで恒久ルール化しない。
- 詳細: [docs/adr/0030-konsen-trio-partner-width-rejected.md](../adr/0030-konsen-trio-partner-width-rejected.md)

### ③ 条件付き単勝（単勝EV≥閾値の馬だけ）の追加（棄却 / 2026-06-21 / ADR 0033）

「`model_win_prob × 単勝オッズ ≥ EV 閾値`（100%/110%/120%）を満たす馬の単勝を¥500 追加/捻出する」案。

- 検証: 71R（2026-05-30, 05-31, 06-13, 06-14）を `scripts/predict-check/win_backtest.py` で評価。
- 結果: **全閾値・全予算モードで baseline を下回った**。

  | variant | 回収率 | 差分 | 単勝発動 |
  |---|---|---|---|
  | baseline | 79.5% | — | — |
  | +win≥100%/add | 76.0% | −3.5pt | 344 回 |
  | +win≥100%/split | 74.5% | −5.0pt | 344 回 |
  | +win≥110%/add | 76.0% | −3.5pt | 340 回 |
  | +win≥110%/split | 74.5% | −5.0pt | 340 回 |
  | +win≥120%/add | 76.0% | −3.5pt | 327 回 |
  | +win≥120%/split | 74.5% | −5.0pt | 327 回 |

- **結論**: 単勝は追加しない。発動頻度が高すぎ（71R で平均 4.6〜4.9 頭）、閾値上げても改善なし。単勝市場はモデルの win prob と高相関でエッジが出なかった。
- 詳細: [docs/adr/0033-conditional-win-bet-rejected.md](../adr/0033-conditional-win-bet-rejected.md)（関連 Issue: #208）

### ④ 上位近接時の2軸フォーメーション（棄却 / 2026-06-27 / ADR 0039）

上位2頭（◎=最大勝率・○=2番手）のモデル勝率が近接した局面で、◎1頭軸ながしを2軸フォーメーションに差し替える案。◎がコケても○×穴で拾えて回収率・的中率が改善するかを検証した（動機: 2026-06-27 福島R7 ③20.7%/⑤19.4%）。発動条件 `prob(○) ≥ θ × prob(◎)` を θ=0.80/0.85/0.90 で掃引。

- 検証: 127R（2026-05-30, 05-31, 06-13, 06-14, 06-20, 06-21、win オッズ・結果が揃う全レース）を `scripts/predict-check/formation_backtest.py` で評価（α=0.2・m=10）。
- 結果: 2 variant とも baseline を上回らなかった。
  - **pair2（◎○両方必須・AND）は全閾値で壊滅**（発動部分集合 ROI 31〜39%・的中 11〜15%）。近接＝拮抗ほど両崩れしやすく、AND 集中は最悪。
  - **union2（◎脚∪○脚に予算折半・OR）は回収率が改善せず**。部分集合 ROI は θ=0.80 で +3.4pt（65.7→69.1）だが θ=0.85 で −5.3pt・θ=0.90 でフラットと符号反転（小標本ノイズ）。的中率は 42%→72〜89% に激増・分散は半減するが、回収が伴わない典型的な「高的中・低配当」。動機の福島R7 が属する θ=0.90 帯でも baseline とフラット。
- **結論**: フォーメーションは採用しない。現行の ◎1頭軸ながしを維持。的中率・分散の改善は回収率を伴わなければ無価値（ADR 0030「相手拡大」棄却と同型）。
- 詳細: [docs/adr/0039-formation-2axis-rejected.md](../adr/0039-formation-2axis-rejected.md)（関連 Issue: #241）

### ⑤ EV ゲート閾値（ROI≥100%）の引き下げ（棄却 / 2026-06-27 / ADR 0040）

「+EV の買い目が全然来ない＝閾値が厳しすぎるのでは」という不安を起点に、2026 年の +EV レース数を調査し、ゲート閾値 ROI≥100% の妥当性を再検証した。

- 検証: DB 保存オッズ＋ `live_ev.py`（全3券種の期待 ROI）で再評価。ワイド込みで厳密評価できたのは 06-21・06-27 の **69R**（+EV **0本**・最高 ROI 97%・**平均 ROI 73.1%**）。α=0.2/0.3 とも +EV 0本。
- 結果: **ROI≥100% は損益分岐そのもの**で、下げる＝−EV を承知で買う＝長期で負け。平均 ROI 73.1% は本戦略が使う券種の JRA 控除率（ワイド・馬連 22.5%／3連複 25%）由来のフェア ROI ≈ 75〜77.5% とほぼ同水準（むしろ僅かに下）で、モデルに平均的なエッジは無い（=平均的に市場と等価。予測 vs 実現の真の較正は別問題で未検証）。+EV は控除率を食い破る稀な歪みだけで構造的に稀＝買い目が来ないのは仕様どおり。閾値直下に「買うべきレース」が溜まる兆候は無し（結果のある 71R の実現 ROI も baseline 83.9%／box-off 84.0%＝ADR 0028 で 100% 未満）。
- **結論**: ROI≥100% を維持。閾値引き下げは不採用。唯一意味のある検証＝推定 ROI の較正（予測 vs 実現）は本検証に足る標本がまだ無い（DB 単一ストアではワイドと結果が同居せず、06-21 の 35R のみ結果を netkeiba 取得すれば予備的に可能だが単日・小標本・非確定オッズで確証不足）→ #248 のデータ蓄積後に #249 で実施（保留）。
- 詳細: [docs/adr/0040-ev-gate-threshold-lowering-rejected.md](../adr/0040-ev-gate-threshold-lowering-rejected.md)（関連 Issue: #248 / #249）

## バックテスト/実績ログ（ADR 未収録）

ADR 化していないが、現行ルール（確率配分・混戦ボックス自動追加）の効果を裏付ける実測記録。数値はセッション時の実測ログに基づく当時の集計値で、再現スクリプトは未整備。

### 検証サマリ: 2026-05-31 主要 6R

配分方式を段階的に変えた時の回収率の比較。

| 配分方式 | 回収率 |
|---|---|
| 手組み ¥2,000/R | 81.7% |
| ¥5,000/R 確率配分 | 94.8% |
| ＋混戦ボックス自動追加 | 103.1%（+¥940） |

- 表の「+¥940」は最終構成（混戦ボックス追加）の実測純益（当時のセッションログ値、6R 合計 ¥30,000 ベース）。各方式の差分ではなく最終構成の絶対損益を指す。
- 白百合S: ◎⑧が着外だったが、混戦ボックスが①④⑦（2,500円）を捕捉し −5,000 → −2,500 に改善。
- 示唆: 確率配分と混戦ボックスはいずれも回収率を押し上げた。

### 実績ログ: 2026-06-20

- 21R 中 +EV は函館12R（◎④1.7倍 / model 35% / ROI〜125%）の **1 レースのみ**。
- 当該レースを ¥5,000 → ¥10,000 に増額し、**回収 222% / +¥12,160**。
- 断然人気（東京10R 1.7倍 80% 等）は全部 −EV で見送り、結果として正解だった。
- 示唆: +EV を厳選して増額する現行方針（レース選択基準）が機能した一例。
