# ブラウザテストケース: ライブ EV 買い目ビュー（LiveBets）

対象: SPA `web/src/routes/LiveBets.tsx`（ルート `/live/:date`）。設計書 [live-ev-buy-view.md](../../docs/specifications/live-ev-buy-view.md) / ADR [0064](../../docs/adr/0064-live-ev-buy-view.md)。

実施は **実装 PR#2（SPA）** で行う（本 PR は設計書のみ）。検証環境は Playwright MCP 不在のため **headless Chrome + puppeteer-core** で代替する（`reference_browser_test_fallback`）。テストデータは `live_ev_snapshots` に隔離 DB へ投入し、テスト後クリーンアップする（`isolated_db_for_write_tests`）。

前提データ（`live_ev_snapshots` に投入する最新サイクルの想定）:
- 函館12R: `verdict='bet'`, ROI 125%, ◎④(model35% 単勝1.7), 混戦なし, フリップなし。
- 東京10R: `verdict='skip'`, ROI 80%, ◎④断然人気, フリップなし。
- 小倉5R: `verdict='skip'`, ROI 78.9%, 前サイクル ROI 103%（`ev_reversed=true`）＋ ◎⑥→⑨（`axis_changed=true`）。

---

### TC-01: 一望サマリ（張るあり）
| 項目 | 内容 |
|------|------|
| 前提 | 上記前提データ（張る 1 本・監視 3 本）が最新サイクルで投入済み |
| 画面 | `/live/2026-06-20` |
| 操作 | 画面を開く |
| 期待結果 | 冒頭に `🟢張る 1レース（監視中 3R）` と最終更新時刻が 1 行で表示される |
| 確認ポイント | 張る本数・監視数・更新時刻が `summary` と一致。前サイクル/朝リストが混在しない |

### TC-06b: フリップ片側のみ真（軸変化なし／EV反転なし）
| 項目 | 内容 |
|------|------|
| 前提 | (a) `ev_reversed=true` かつ `axis_changed=false`（◎不変で +EV→−EV）／(b) `axis_changed=true` かつ `ev_reversed=false`（軸だけ変わり verdict は不変）の 2 レースを投入 |
| 画面 | `/live/2026-06-20` |
| 操作 | 各レースの表示を確認 |
| 期待結果 | (a) は「+EV→−EV 反転」のみ、(b) は「◎変化」のみを強調し、片側 false の事象を誤って強調しない |
| 確認ポイント | `axis_changed`/`ev_reversed` が独立に UI 強調へ反映され、両真のみを前提にした実装になっていない |

### TC-07: 最新サイクルのみ表示（時系列の混入なし）

### TC-10: 混戦だが印馬≤2 でボックス不成立（ながし転用）
| 項目 | 内容 |
|------|------|
| 前提 | `konsen=true` だが印馬が 2 頭以下で 3連複ボックスが組成不能（CLAUDE.md「印馬が 2 頭以下はボックス不成立→ながし¥1,500 を◎軸ながしに転用」）。slip に `method='box'` leg が無く、3連複◎軸ながしに配分が寄っている |
| 画面 | `/live/2026-06-20` |
| 操作 | 当該レースの 3連複表示を確認 |
| 期待結果 | ボックス（方式=ボックス）は表示されず、3連複は◎軸ながしのみ。金額配分が転用先（ながし）に寄る |
| 確認ポイント | box 不成立時に空のボックス欄や¥0 の box leg を出さない。混戦配分の境界が正しく描画される |
