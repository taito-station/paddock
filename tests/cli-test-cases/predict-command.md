# CLI テストケース: predict コマンド

`paddock-analyze predict <race_id>` の動作確認手順。

## TC-01: 正常ケース（全馬にスタッツあり）

| 項目 | 内容 |
|------|------|
| 前提 | DBに race_card と horse/course/jockey スタッツが十分に蓄積されている |
| コマンド | `paddock-analyze predict <有効な race_id>` |
| 期待結果 | 馬番・馬名・勝率・連対率・複勝率の表が表示される |
| 確認ポイント | 各確率が 0.0〜1.0 の範囲内 / 勝率の合計が ≒ 1.0 / 連対率の合計が ≒ 1.0 / 複勝率の合計が ≒ 1.0 |

## TC-02: 存在しない race_id を指定

| 項目 | 内容 |
|------|------|
| 前提 | 指定する race_id が DB に存在しない |
| コマンド | `paddock-analyze predict 9999999999` |
| 期待結果 | `not found` 等のエラーメッセージが stderr に出力され、exit code 1 |
| 確認ポイント | パニックせずにエラーハンドリングされること |

## TC-03: スタッツ件数ゼロの馬が含まれる（個別ゼロスコア）

| 項目 | 内容 |
|------|------|
| 前提 | 出走馬のうち 1 頭以上の過去成績が 0 件、他の馬にはスタッツあり |
| コマンド | `paddock-analyze predict <対象 race_id>` |
| 期待結果 | エラーにならず全馬の確率が表示される |
| 確認ポイント | スタッツなし馬の win_prob / place_prob / show_prob は 0.0% と表示 / **全馬を含む** win_prob 合計が ≒ 1.0（スコアゼロ馬は win_prob=0 で正規化の分母に含まれるため総和は 1.0 になる） |

## TC-03b: 全馬スタッツ件数ゼロ（均等フォールバック）

| 項目 | 内容 |
|------|------|
| 前提 | 全出走馬の過去成績が 0 件（未蓄積の新コース・新シーズン等） |
| コマンド | `paddock-analyze predict <対象 race_id>` |
| 期待結果 | 均等確率（1/頭数）で全馬が表示される |
| 確認ポイント | 全馬の win_prob が ≒ 1/頭数 / 合計確率が ≒ 1.0 |

## TC-04: 騎手なし（jockey = None）の馬が含まれる

| 項目 | 内容 |
|------|------|
| 前提 | HorseEntry の jockey フィールドが None の馬がいる |
| コマンド | `paddock-analyze predict <対象 race_id>` |
| 期待結果 | 騎手スタッツなしでスコア計算され、結果が表示される |
| 確認ポイント | 騎手なしでもクラッシュしないこと |

## TC-05: race_card が未保存の race_id を指定

| 項目 | 内容 |
|------|------|
| 前提 | races テーブルには存在するが race_card_entries が未保存の race_id |
| テスト準備 | `paddock-parse-pdf` で race を保存した後、`paddock-parse-entries` は実行しない。または `race_card_entries` テーブルにデータが存在しない race_id を用意する |
| コマンド | `paddock-analyze predict <race_id>` |
| 期待結果 | `race card not found` 等のエラーが stderr に表示される |
| 確認ポイント | 空の結果セットが返らず、`race card not found` に相当するエラーメッセージが stderr に表示されること / exit code が 1 であること |

## EV・Kelly 配分テストケース（Issue #12）

### TC-06: EV 計算・買い目推奨の正常表示

| 項目 | 内容 |
|------|------|
| 前提 | DB に race_card / スタッツ / オッズが存在し、馬連の EV > 1.0 となる推奨が少なくとも 1 件ある。BettingConfig::default()（ev_threshold=1.0, kelly_cap=0.25）を使用 |
| コマンド | `paddock-analyze predict <有効な race_id>` |
| 期待結果 | 確率表に加えて「買い目推奨」セクションが表示される |
| 確認ポイント | 表示される全買い目の `ev = probability × odds` が `ev_threshold (1.0)` を超えていること / 馬連の推奨が馬単・三連複・単勝・複勝・三連単の推奨よりも先に表示される（優先度順） / EV フィルタ通過後は kelly_fraction > 0 かつ ≤ kelly_cap (0.25) であること |

### TC-07: EV が全買い目で閾値未満のレース

| 項目 | 内容 |
|------|------|
| 前提 | スタッツはあるが、全買い目の EV が ev_threshold (1.0) 以下（EV = 1.0 ちょうどを含む。strict > のため EV = 1.0 は除外される） |
| コマンド | `paddock-analyze predict <対象 race_id>` |
| 期待結果 | 「買い目推奨なし」等のメッセージが表示される |
| 確認ポイント | エラーにならず正常終了 / 確率表自体は表示される |

### TC-08: 三連単が trifecta_ev_threshold (2.0) を超える場合のみ出力

| 項目 | 内容 |
|------|------|
| 前提 | 特定の三連単組み合わせのオッズが非常に高い（EV > 2.0）、かつ馬連・馬単・三連複のいずれかが少なくとも 1 件 EV > 1.0 であること |
| コマンド | `paddock-analyze predict <対象 race_id>` |
| 期待結果 | 三連単が買い目推奨に含まれる |
| 確認ポイント | 三連単の EV が 2.0 を超えていること（strict > のため EV = 2.0 ちょうどは除外） / 馬連・馬単・三連複・単勝・複勝の後に表示されること（優先度順：最後尾） |

### TC-09: オッズ未取得のレース（RaceOdds 空）

| 項目 | 内容 |
|------|------|
| 前提 | race_card は保存済みだがオッズスクレイパーがまだ実行されていない |
| コマンド | `paddock-analyze predict <対象 race_id>` |
| 期待結果 | 確率表は表示され、買い目推奨セクションは「オッズ未取得」等のメッセージを出す |
| 確認ポイント | パニックしないこと / 確率推定自体は完了すること |

## predict 対話セッションテストケース（Issue #13）

> **共通前提**: 特記なき限り `samples/2026-3nakayama6.pdf` を取り込んだ DB 状態を基準とする。取り込みにより `races` テーブル（`find_races_by_date` の対象）と `race_card` の両方にレコードが入る。
> オッズが必要なケースは、対象 race_id の `race_odds` を保存済みとする（`race_odds` テーブル追加前は手動 INSERT またはモックで代替）。
> 「買い目推奨あり」を要求するケース（TC-12 等）では、`select_bets` の EV 閾値（馬連等 1.0 / 三連単 2.0、strict `>`）を超える推奨が必要本数だけ出るオッズ・確率データを用意する（手動 INSERT したオッズで EV を調整する）。

### TC-10: 正常セッション（y 選択 → 払い戻しあり）

| 項目 | 内容 |
|------|------|
| 前提 | 指定日に 2 件以上のレースが DB に存在し、全レースの race_card とオッズが保存済み |
| コマンド | `paddock-predict --date <開催日> --budget 10000` |
| 操作 | 各レースで `y` を入力して推奨通りに購入 → 払い戻し額を入力 |
| 期待結果 | レースごとに確率表・買い目推奨が表示され、払い戻し入力後に残高が更新される |
| 確認ポイント | 推奨額の合計が残高を超えないこと（残高ガード） / 残高が「初期予算 − 賭け金合計 + 払戻合計」と一致すること / 全レース終了後に一日集計が表示され、P&L = 総払戻 − 総賭け金 と一致すること |

### TC-11: 全レーススキップ（s 選択）

| 項目 | 内容 |
|------|------|
| 前提 | 指定日に 1 件以上のレースが存在する |
| コマンド | `paddock-predict --date <開催日> --budget 10000` |
| 操作 | 全レースで `s` を入力 |
| 期待結果 | 買い目推奨が表示されるが賭け金は発生せず、残高が初期予算のまま維持される |
| 確認ポイント | 総賭け金 = ¥0 / 最終残高 = 初期予算 / 一日集計が正しく表示されること |

### TC-12: 金額編集モード（e 選択）

| 項目 | 内容 |
|------|------|
| 前提 | 買い目推奨が 2 件以上表示されるレースが存在する（EV 閾値を超える推奨が 2 件以上出るオッズ・確率データが DB にある状態） |
| コマンド | `paddock-predict --date <開催日> --budget 10000` |
| 操作 | `e` を入力して一部の買い目に任意金額・一部に `0` を入力 |
| 期待結果 | `0` を入力した買い目はスキップされ、他の買い目のみ賭け金が計上される |
| 確認ポイント | 合計賭け金が入力した金額の合計と一致すること / 残高が正しく更新されること |

### TC-13: 開催なし日付の指定

| 項目 | 内容 |
|------|------|
| 前提 | 指定日の races テーブルに該当レースが存在しない |
| コマンド | `paddock-predict --date 2000-01-01 --budget 10000` |
| 期待結果 | 「この日の開催はありません」等のメッセージが **stdout** に表示されて正常終了 |
| 確認ポイント | exit code 0 で終了すること / stderr へのエラー出力がないこと |

### TC-14: オッズ未取得レースを含むセッション

| 項目 | 内容 |
|------|------|
| 前提 | 複数レースのうち 1 件のみオッズが未取得（find_race_odds が None） |
| コマンド | `paddock-predict --date <開催日> --budget 10000` |
| 操作 | オッズ未取得レースに到達する |
| 期待結果 | 「オッズ未取得 — このレースはスキップします」が表示され、`[s]` のみ受け付ける（`y` / `e` は提示されない）。他のレースは正常に処理される |
| 確認ポイント | 未取得レースでパニックしないこと / 未取得レースの賭け金が ¥0 であること / セッション全体が継続すること |

### TC-15: 払い戻しなし（Enter のみ）

| 項目 | 内容 |
|------|------|
| 前提 | オッズ・推奨があるレースが存在する |
| コマンド | `paddock-predict --date <開催日> --budget 10000` |
| 操作 | `y` で購入後、払い戻し入力で何も入力せず Enter のみ押下 |
| 期待結果 | payout = ¥0 として扱われ、残高が賭け金分だけ減少する |
| 確認ポイント | 残高 = 直前残高 − 賭け金 となること / 損益がマイナス表示されること |

### TC-16: 残高超過（推奨額 > 残高）

| 項目 | 内容 |
|------|------|
| 前提 | 残高が小さく（例: `--budget 100`）、推奨額合計が残高を超えるレースがある |
| コマンド | `paddock-predict --date <開催日> --budget 100` |
| 操作 | `y` を選択して残高を超える推奨額を確定しようとする |
| 期待結果 | 残高超過の旨が表示され、確定できず `e`（編集）または `s`（スキップ）に誘導される |
| 確認ポイント | 残高が 0 未満にならないこと / パニック（u64 underflow）しないこと |
