# CLI テストケース: backtest コマンド

`paddock-analyze backtest --from YYYY-MM-DD --to YYYY-MM-DD` の動作確認手順。

> **共通前提**: 特記なき限り `samples/2026-3nakayama6.pdf` 等を取り込み、`races`/`results` に
> 確定済み（`finishing_position` あり・`source='pdf'`）レースが複数日分蓄積された DB 状態を基準とする。

## TC-01: 正常ケース（期間内に確定レースあり）

| 項目 | 内容 |
|------|------|
| 前提 | 指定期間に確定済みレースが複数存在し、各馬の過去成績も蓄積されている |
| コマンド | `paddock-analyze backtest --from <開始日> --to <終了日>` |
| 期待結果 | 評価レース数・単勝/連対/複勝的中率・想定回収率・Brier・LogLoss のレポートが表示される |
| 確認ポイント | 各的中率が 0.0〜1.0（0〜100%）の範囲内 / 単勝的中率 ≤ 連対的中率 ≤ 複勝的中率（包含関係） / Brier・LogLoss が非負 / `races_evaluated` が「突合できた（エントリ 1 頭以上でトップ選好馬を決定できた）レース数」と一致し、取得レース数以下であること |

## TC-02: 期間内に確定レースが無い

| 項目 | 内容 |
|------|------|
| 前提 | 指定期間の `races` に `source='pdf'`・`finishing_position` ありのレースが存在しない |
| コマンド | `paddock-analyze backtest --from 2000-01-01 --to 2000-12-31` |
| 期待結果 | 「評価対象レースなし」等のメッセージが stdout に表示され、exit code 0 で正常終了 |
| 確認ポイント | パニックしないこと / ゼロ除算で NaN を出力しないこと（評価レース数 0 を明示） |

## TC-03: オッズ欠落レースを含む（回収率の母数除外）

| 項目 | 内容 |
|------|------|
| 前提 | 期間内の確定レースのうち一部で `results.odds` が NULL（オッズ未取り込み） |
| コマンド | `paddock-analyze backtest --from <開始日> --to <終了日>` |
| 期待結果 | 的中率・Brier・LogLoss は全評価レースで算出され、回収率はオッズ取得できたレースのみを母数に算出される |
| 確認ポイント | `payout_races` < `races_evaluated` となること / 回収率の母数が 0 件のときは回収率が「—」等で表示され NaN にならないこと / オッズ欠落でクラッシュしないこと |

## TC-04: リーク防止（as-of カットオフ）の検証

| 項目 | 内容 |
|------|------|
| 前提 | ある馬が期間内レース R（レース日 D）に出走し、かつ D 当日・D 以降にも複数の好成績がある |
| 検証方法 | R 単体を対象に backtest を実行した結果と、`predict <R の race_id>`（全期間統計）の確率を比較する。または gateway の as-of 集計を単体/結合テストで直接検証する |
| 期待結果 | backtest（as-of=D）では D 以降の成績が当該馬の統計に**含まれない**ため、`predict`（全期間）より当該馬の確率が低く（または異なって）出る |
| 確認ポイント | D 当日・D 以降の `results` が as-of 統計に混入していないこと / 同一開催日（D）内の他レース結果も統計に寄与しないこと |

## TC-05: from > to（不正な期間指定）

| 項目 | 内容 |
|------|------|
| 前提 | 開始日が終了日より後 |
| コマンド | `paddock-analyze backtest --from 2026-03-31 --to 2026-01-01` |
| 期待結果 | 結果が空集合になり「評価対象レースなし」を stdout に表示して exit code 0 で正常終了（期間の前後関係はバリデーションせず空集合扱い） |
| 確認ポイント | パニックしないこと / 負の件数・NaN を出力しないこと / exit code 0 |

## TC-06: 日付フォーマット不正

| 項目 | 内容 |
|------|------|
| 前提 | `--from` / `--to` に不正な日付文字列を渡す |
| コマンド | `paddock-analyze backtest --from 2026/01/01 --to 03-31` |
| 期待結果 | 日付パースエラーが stderr に表示され、exit code 1 |
| 確認ポイント | パニックせずにエラーハンドリングされること |
